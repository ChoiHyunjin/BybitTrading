# Sprint 1: UX 설계 - 핵심 모델 & 지표 엔진

## 범위
Sprint 1은 UI가 없는 데이터 모델/엔진 레이어. UX 관점에서 향후 UI에 필요한 데이터 구조 요구사항을 정의한다.

## 데이터 표시 요구사항 (향후 UI용)

### Trade 정보 표시
- 매수/매도 구분 (BUY/SELL)
- 거래 시점 (타임스탬프 → 사람이 읽을 수 있는 형태)
- 거래 가격, 수량
- 개별 거래 수익/손실 (금액 + 퍼센트)
- 수익: 초록, 손실: 빨강 (색상 구분용 부호 필요)

### BacktestResult 요약 표시
- 총 수익률 (%)
- 승률 (%)
- 최대 낙폭 MDD (%)
- 총 거래 수
- 총 수익/손실 금액

### 지표 데이터 (차트용)
- 볼린저밴드 상단/하단/중심선
- 단기/장기 이동평균
- 밴드폭 (BW)

## 상태 정의
- `idle`: 초기 상태, 모델 미생성
- `calculating`: 지표 계산 중
- `complete`: 계산 완료, 결과 사용 가능

## UX Copy 가이드
- 수익률: "+12.5%" / "-3.2%" (부호 필수)
- 승률: "65.0%"
- MDD: "-15.3%" (항상 음수)
- 거래 수: "23건"
